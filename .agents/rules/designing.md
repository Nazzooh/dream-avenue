# Designing Skill Rules

## Objective
Create interfaces that are:
- Modern (aligned with current UI/UX trends)
- Premium (clean, high-end feel, no clutter)
- Clear (user instantly understands purpose)

## Core Principles
1. Clarity over creativity
2. Consistency across all screens
3. Minimalism (remove unnecessary elements)
4. Accessibility (readable fonts, contrast, spacing)

## UI Requirements
- Use consistent color palette (max 3–5 core colors)
- Typography must be readable (no decorative overload)
- Spacing must follow a system (8px grid recommended)
- Use components (buttons, cards, inputs) consistently

## UX Requirements
- Every screen must answer: "What should the user do here?"
- Navigation must be obvious (no guessing)
- Reduce steps required to complete tasks
- Provide feedback (loading, success, error states)

## Design Process
1. Understand project goal
2. Define user flow
3. Create wireframe
4. Apply visual design
5. Validate usability

## Strict Rules
- No random design decisions
- No overcrowded UI
- No inconsistent spacing or fonts
- No unclear actions

## Output Expectation
- Clean layout
- Clear hierarchy
- Professional look comparable to top-tier apps
